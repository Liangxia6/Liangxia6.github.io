## 什么是 Agent？

AI Agent（人工智能代理）是一个能够**自主感知环境、做出决策并执行行动**的智能系统。它不仅仅是"问一句答一句"的聊天机器人，而是能够：

- 理解用户意图
- 拆解复杂任务
- 调用工具执行操作
- 观察执行结果并调整策略
- 记住跨会话的信息

## Agent ≠ LLM

这是理解 Agent 架构的第一个关键概念：

| 组件 | 角色 | 类比 |
|------|------|------|
| **LLM**（大语言模型） | 推理引擎、大脑 | 人的思维能力 |
| **Agent 框架** | 执行系统、身体 | 人的手脚和感官 |

LLM 负责"思考"，Agent 框架负责"行动"。两者通过 **ReAct 循环**（Reasoning + Acting）协作。

## ReAct 循环的核心流程

一次完整的 Agent 周转（从用户输入到最终回复）包含以下步骤：

```
用户输入
    │
    ▼
① Prompt 装配
    ├─ 加载系统角色设定（Persona）
    ├─ 注入持久记忆（Memory）
    ├─ 加载专业知识（Skills）
    ├─ 注册可用工具（Tools）
    └─ 拼接对话历史（History）
    │
    ▼
② LLM 推理（第1次调用）
    ├─ 分析用户意图
    ├─ 决定：输出文本 或 调用工具？
    │
    ├── 如果是"调用工具" ────────────┐
    │   ↓                            │
    │ ③ Agent 框架执行工具           │
    │   ├─ terminal() → 执行命令     │
    │   ├─ read_file() → 读文件      │
    │   └─ 结果返回 LLM              │
    │   ↓                            │
    │ ④ LLM 再次推理（第2次调用）    │
    │   ├─ 查看工具执行结果           │
    │   └─ 再次决定...               │
    │   └─ 循环直到不再调用工具       │
    │                                │
    └── 如果是"输出文本" ←───────────┘
        ↓
    ⑤ 最终响应送达用户
```

**关键洞察**：一轮用户请求可能需要 **多次 LLM 调用**。我最高记录是在一次任务中调用了 7 次 LLM（多次工具调用 + 结果分析）。

## 实际案例：用 Hermes Agent 下载代码

以下是我和 Hermes Agent 交互的一个真实过程：

```
用户请求："下载hermes项目代码到桌面"
    │
LLM 调用 #1：分析意图 → 决定用 git clone
    │
Agent 执行：terminal("git clone https://github.com/nousresearch/hermes-agent.git")
    │
LLM 调用 #2：看到 exit_code=0 → 需要验证文件是否存在
    │
Agent 执行：terminal("ls && du -sh")
    │
LLM 调用 #3：确认成功 → 输出最终回复
    │
完成！共 3 次 LLM 调用，2 次工具执行
```

## 为什么这个区分很重要？

1. **性能优化**：每次 LLM 调用都消耗 tokens 和时间。好的 Agent 框架会尽量减少不必要的 LLM 调用
2. **成本控制**：LLM API 按 token 计费，减少调用次数就是降低成本
3. **架构理解**：区分"思考"和"行动"是设计好 Agent 的关键
